`from contextlib import contextmanager` 是 Python 提供的一种**简化自定义上下文管理器**（`with` 语句）的方式。  
## 1. 作用 
* 让你用 **函数** 的形式实现上下文管理，而不需要写一个完整的类（实现 `__enter__` / `__exit__`）。
* 典型用途：

  * 资源管理（文件、数据库连接、锁等）
  * 临时修改全局状态，执行任务后恢复
  * 临时代码环境（比如切换目录、捕获异常、计时等）

---

## 2. 基本语法

```python
from contextlib import contextmanager

@contextmanager
def my_context():
    # 前置逻辑（进入 with 时执行）
    yield some_value
    # 后置逻辑（退出 with 时执行）
```

说明：

1. **`yield` 之前** 的代码相当于 `__enter__` 方法。
2. **`yield`** 的值会传给 `with ... as var` 的 `var`。
3. **`yield` 之后** 的代码相当于 `__exit__` 方法，不管 `with` 代码块是否抛异常都会执行。
4. 如果需要捕获异常，可以用 `try/finally` 或 `try/except` 包裹 `yield`。

## 3. 简单示例
### 例 1：计时器

```python
import time
from contextlib import contextmanager

@contextmanager
def timer(name):
    start = time.time()
    try:
        yield  # 进入 with 块
    finally:
        end = time.time()
        print(f"{name} 耗时: {end - start:.4f} 秒")

# 使用
with timer("任务1"):
    time.sleep(1.2)
```

输出：

```
任务1 耗时: 1.2005 秒
```

### 例 2：临时切换工作目录

```python
import os
from contextlib import contextmanager

@contextmanager
def change_dir(path):
    old_dir = os.getcwd()
    os.chdir(path)
    try:
        yield
    finally:
        os.chdir(old_dir)  # 确保退出时恢复

# 使用
print("原目录:", os.getcwd())
with change_dir("/tmp"):
    print("临时目录:", os.getcwd())
print("恢复目录:", os.getcwd())
```

### 例 3：模拟文件打开（简化版）

```python
from contextlib import contextmanager

@contextmanager
def open_file(path, mode):
    f = open(path, mode)
    try:
        yield f
    finally:
        f.close()

with open_file("test.txt", "w") as f:
    f.write("Hello, World!")
```

这里的行为等同于 `with open("test.txt", "w") as f:` 

## 4. 使用场景总结

* 替代完整类写法的上下文管理器
* 代码结构更紧凑、可读性好
* 能确保即使中途异常也能执行清理逻辑（如关闭文件、释放资源、恢复状态）
